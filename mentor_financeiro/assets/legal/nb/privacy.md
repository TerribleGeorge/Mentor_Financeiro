# Personvernerklæring - Mentor Financeiro

**Sist oppdatert:** 19. mai 2026

**Søknad:** Mentor Financeiro

**Pakke Android:** `com.georgeguimares.mentorfinanceiro`

**Retningslinjer publisert på:** https://terriblegeorge.github.io/Mentor_Financeiro/

Denne personvernerklæringen forklarer tydelig hvilke data Mentor Financeiro kan få tilgang til, samle inn, bruke, lagre, dele og beskytte. Den er skrevet for å gjenspeile den nåværende driften av applikasjonen og bør leses sammen med vilkårene for bruk og sikkerhetssiden.

Mentor Financeiro er et program for finansiell organisering, finansiell utdanning, mål, utgiftsposter, grafer, rapporter, investeringsinnhold og valgfrie Premium abonnementsfunksjoner. Søknaden er ikke en bank, megler, finansinstitusjon, registrert investeringsrådgiver, forsikringsselskap eller betalingsmåte.

Mentor Financeiro tilbys av utvikleren som er ansvarlig for å publisere applikasjonen på Google Play.

For å utøve rettigheter knyttet til dataene dine, stille spørsmål eller be om sletting, kontakt oss på e-post:

**george.guimares@gmail.com**

## 2. Sammendrag i klart språk

– Appen hjelper deg med å organisere utgifter, mål, grenser, økonomisk historie, investeringer og pedagogisk innhold.
- For å bruke visse funksjoner kan du logge på med e-post/passord, Google-konto, Apple-konto eller inkognitomodus, gjennom Firebase Authentication.
- De økonomiske dataene du oppgir, som utgifter, mål, grenser, profil og preferanser, kan lagres i Firebase/Cloud Firestore for synkronisering og kontodrift.
- Appen bruker Google/Firebase-tjenester for pålogging, database, push-varslinger, analyser og feildiagnose.
- Appen kan bruke Google AdMob for å vise annonser.
- Premium behandles av Google Play Billing i selve appen.
– Plassering er valgfritt og brukes kun når du godkjenner det, for eksempel for å tilpasse investeringssammenligninger til landet eller den lokale konteksten.
- Varslingsovervåkingsfunksjonen er valgfri. Hvis du aktiverer varslingstilgang på Android, kan appen lese tittelen og teksten til støttede varsler for å identifisere kjøp, betalinger og overføringer og gjøre disse om til økonomiske poster. Appen får ikke tilgang til bankkontoen din og ber ikke om passord.
- Bilder valgt av deg for personalisering kan lagres lokalt på enheten.
- Du kan be om sletting av konto og data via e-posten som er angitt i denne policyen.

## 3. Data vi kan behandle

Avhengig av hvordan du bruker applikasjonen, kan vi behandle følgende kategorier av data.

### 3.1 Konto- og identifikasjonsdata

- Navn oppgitt i registreringen.
- E-post.
- Firebase (UID) brukeridentifikator.
- Påloggingsmetode brukt.
- Profilbilde, når levert av påloggingsleverandøren.
- Premium abonnementsstatus knyttet til profilen.
- Tekniske data som er nødvendige for å opprettholde økten og synkronisere informasjon.

### 3.2 Finansielle data angitt av deg

- Utgifter, utgifter, inntekter, kategorier, finansiell historie og transaksjoner.
- Mål, grenser, planlegging, økonomiske drømmer og preferanser.
- Informasjon brukt i grafer, rapporter, analyser og automatisk veiledning i appen.
- Investorprofil, mål og svar gitt i spørreskjemaer eller planleggingsskjermer.

Disse dataene kan være sensitive i praktisk forstand, fordi de avslører aspekter ved ditt økonomiske liv. Derfor bør du kun registrere opplysninger du ønsker å beholde i søknaden.

### 3.3 Abonnements- og kjøpsdata

- Kjøpt produkt, abonnementsstatus, periode, fornyelse, kjøpsrestaurering og tekniske identifikatorer som kreves for å bekrefte Premium.
- Kjøp, kanselleringer, fornyelser, refusjoner og belastninger behandles av Google Play, i samsvar med regler for Google-konto og butikk.

Appen samler ikke inn eller lagrer kortnumre, kortsikkerhetskoder eller bankbetalingsdetaljer.

### 3.4 Bruk, diagnostikk og ytelsesdata- Bruks- og interaksjonshendelser i appen, for eksempel skjermer som er åpnet, ressurser brukt og fullførte flyter.
- Aggregerte eller tekniske data for å forstå produktstabilitet, ytelse og forbedring.
- Feilrapporter og diagnostikk av Firebase Crashlytics, når det er aktuelt, inkludert teknisk informasjon om enheten, appversjon, operativsystem, feillogger og brukeridentifikasjon når konfigurert.

### 3.5 Push-varslingsdata

Hvis du tillater varsler fra appen, kan vi håndtere:

- Firebase Cloud Messaging token (FCM).
- Preferanser knyttet til mottak av varsler.
- Emner eller segmenter som er nødvendige for å sende kommunikasjon fra selve applikasjonen.

Du kan kontrollere om varsler er tillatt i Android-innstillingene.

### 3.6 Varslingsdata lest i Android

Varslingsovervåking er en valgfri funksjon. Det fungerer bare hvis du manuelt aktiverer varslingstilgang i Android-innstillingene.

Når den er aktivert, kan appen motta tittel, tekst, kildeapppakkenavn, tid og tekniske metadata fra varsler som vises på enheten din. Målet er å identifisere varsler som er kompatible med utgifter, kjøp, betalinger, abonnementer og overføringer for å generere økonomiske poster.

Viktige mål for denne funksjonen:

– Appen får ikke tilgang til bankkontoen din.
– Appen logger ikke inn på banker, kort eller digitale lommebøker.
– Appen ber ikke om bankpassord.
- Appen bruker filtre for å unngå å fange opp sikkerhetskoder, OTP, passord og autentiseringsmeldinger.
– Filterets fokus er å identifisere kompatible kjøp, betalinger, overføringer sendt og økonomiske bevegelser.
- Du kan deaktivere funksjonen i appen eller tilbakekalle tillatelsen i Android-innstillingene.

Selv med filtre kan varsler inneholde sensitiv informasjon. Aktiver denne funksjonen bare hvis du forstår og godtar denne typen behandling.

### 3.7 Plassering

Appen kan be om omtrentlig eller nøyaktig plassering, i henhold til tillatelsen gitt av Android, for å tilpasse sammenligninger, innhold eller investeringskontekst til landet/regionen.

Plassering er ikke nødvendig for alle funksjoner. Hvis du nekter tillatelse, kan noen sammenligninger eller tilpasninger være begrenset.

### 3.8 Lokale bilder og filer

Når du velger et bilde for å visuelt tilpasse appen, kan det lagres lokalt på enheten. Med mindre de er spesifikt implementert i fremtiden, sendes ikke disse personaliseringsbildene til våre servere.

### 3.9 Annonsering

Appen kan vise annonser gjennom Google AdMob. Google kan behandle enhetsidentifikatorer, bruksdata, teknisk informasjon og reklamesignaler i samsvar med Googles egne retningslinjer og brukerens annonseinnstillinger.

### 3.10 Data fra eksterne markeds-APIer

Appen kan konsultere eksterne APIer for valutakurser, sitater, indekser eller markedsinformasjon. Disse spørringene involverer vanligvis parametere som valuta, aktiva, land eller symbol som det søkes i. Det er ikke nødvendig å sende dine registrerte personlige økonomiske data for å få denne informasjonen.

- Vis annonser der det er aktuelt.

## 6. Deling med tredjeparter

Vi selger ikke dine personopplysninger.

Vi kan dele eller tillate behandling av data fra leverandører som er nødvendige for driften av appen, inkludert:

- **Markeds-, valuta- eller tilbuds-APIer**: konsultasjon av offentlig eller markedsøkonomisk informasjon brukt i appen.

Disse tredjepartene kan behandle data i samsvar med deres egne vilkår, retningslinjer og innstillinger.

Vi oppbevarer data så lenge det er nødvendig for å levere applikasjonen, oppfylle formålene beskrevet i denne policyen, overholde juridiske forpliktelser, løse tvister, forhindre svindel og opprettholde sikkerheten.

Generelt:

- Lokale data kan forbli på enheten til du sletter, sletter appdata eller avinstallerer appen.

- Kjøpsinformasjon kan forbli i Google Play i henhold til skatte-, kommersielle og butikkregler.

## 9. Sikkerhet

Vi vedtar rimelige tekniske og organisatoriske tiltak for å beskytte data, inkludert:

- Autentisering av Firebase.
- Tilgangsregler i databasen for å begrense data til den tilsvarende brukeren.
- Kommunikasjon kryptert av SDK-ene og API-ene som brukes.
- Bruk av Android-tillatelser kun når det er relatert til appressurser.
- Filtre ved overvåking av varsler for å unngå sikkerhetskoder, OTP, passord og autentiseringsmeldinger.
- Feildiagnose for å rette feil og forbedre stabiliteten.

Ingen systemer er 100 % sikre. Du bør også beskytte enheten din, bruke en skjermlås, ha et sterkt passord, unngå å dele kontoen din og oppdatere appen når en ny versjon slippes.
